# Avito Parser — мониторинг краткосрочной аренды

Программа для автоматического сбора объявлений краткосрочной аренды с Avito, извлечения данных о ценах, календаре занятости и характеристиках объектов с экспортом результатов в Excel.

---

## Что делает программа

Программа работает в два этапа:

**Этап 1 — Парсинг.** Открывает браузер, переходит по ссылке на категорию посуточной аренды Avito (с уже настроенными фильтрами), обходит все страницы пагинации и собирает базовые данные каждого объявления. Уже на этапе каталога программа извлекает наличие бейджа «Мгновенная бронь» и рейтинг хоста. Затем заходит в каждую карточку объявления и извлекает детальную информацию: координаты объекта, календарь занятости на 60 дней (через интерактивный datepicker), реальные цены для каждого свободного дня (через датепикер: выбор дат заезда и выезда → чтение динамической цены), минимальный срок аренды (кликом на свободную дату в календаре). При большом количестве карточек программа автоматически ротирует прокси для обхода блокировок. Если рейтинг или мгновенное бронирование не были найдены в каталоге — программа ищет их на странице карточки.Все данные сохраняются в локальную базу SQLite.

**Этап 2 — Экспорт.** Формирует Excel-файл с одним листом, где все объявления собраны в таблицу с автофильтрами. Вы можете отфильтровать объявления по категории жилья, отсортировать по средней цене или проценту занятости и сравнить объекты между собой.

- **Python** 3.11 или выше

- **Стабильное интернет-соединение** для парсинга Avito

## Установка

### 1. Клонируйте или скачайте проект

Поместите папку `avito_parser` в удобное место на вашем компьютере.

### 2. Откройте терминал в папке проекта

```bash
cd путь/к/avito_parser
```

### 3. Создайте виртуальное окружение (рекомендуется)

```bash
python -m venv venv
```

Активируйте его:

**Windows:**
```bash
venv\Scripts\activate
```

**macOS / Linux:**
```bash
source venv/bin/activate
```

### 4. Установите зависимости

Основные зависимости:
```bash
pip install -e .
```

Для разработки (линтер, типы, тесты):
```bash
pip install -e ".[dev]"

playwright install chromium
```

Эта команда скачает Chromium (~150 МБ). Загрузка выполняется один раз.

## Настройка

### 1. Создайте файл `.env`

Скопируйте шаблон:

**Windows:**
```bash
copy .env.example .env
```

**macOS / Linux:**
```bash
cp .env.example .env
```

### 2. Заполните обязательные переменные

Откройте файл `.env` в любом текстовом редакторе и заполните:

```env
# Ссылка на категорию посуточной аренды Avito с нужными фильтрами
AVITO_CATEGORY_URL=https://www.avito.ru/sankt-peterburg/kvartiry/sdam/posutochno
```

### 3. Настройте необязательные параметры (при необходимости)

```env
# Запуск браузера без графического интерфейса (true/false)
# false — вы увидите окно браузера, true — браузер работает в фоне
HEADLESS_MODE=false

# Максимальное количество страниц для парсинга (0 = все страницы)
MAX_PAGES=5

# Путь к файлу со списком прокси (формат: host:port:user:pass, по одному на строку)
# Если пусто — парсер работает без прокси (с вашим IP)
PROXY_FILE_PATH=data/proxies.txt

# Менять прокси каждые N карточек (0 = только при бане)
ROTATE_EVERY_N_LISTINGS=70

# Уровень логирования (DEBUG — максимум деталей, INFO — стандартный)
LOG_LEVEL=INFO

# Путь к файлу Excel с результатами
EXPORT_PATH=data/avito_report.xlsx
```

### Описание всех переменных окружения

| Переменная | Обязательная | По умолчанию | Описание |
|---|---|---|---|
| `AVITO_CATEGORY_URL` | Да | — | URL категории посуточной аренды Avito с фильтрами |
| `HEADLESS_MODE` | Нет | `false` | Режим без графического интерфейса |
| `NAVIGATION_TIMEOUT` | Нет | `90000` | Таймаут навигации (мс) |
| `PAGE_WAIT_TIME` | Нет | `30000` | Ожидание после загрузки страницы (мс) |
| `MAX_PAGES` | Нет | `5` | Лимит страниц пагинации (0 = все) |
| `DB_PATH` | Нет | `data/avito_listings.db` | Путь к базе данных SQLite |
| `EXPORT_PATH` | Нет | `data/avito_report.xlsx` | Путь к выходному Excel-файлу |
| `LOG_LEVEL` | Нет | `INFO` | Уровень логирования |
| `LOG_FILE_PATH` | Нет | — | Путь к файлу логов (пусто = только консоль) |
| `ROTATE_EVERY_N_LISTINGS` | Нет |	70 | Менять прокси каждые N карточек (0 = только при бане) |
| `PROXY_FILE_PATH` |	Нет |	— |	Путь к файлу со списком прокси (пусто = без прокси) |

## Запуск

### Основная команда

```bash
python -m src
```

### Что происходит при запуске

1. Программа загружает настройки из `.env` и проверяет обязательные переменные.
2. Открывается браузер Chromium со stealth-настройками (обход антибот-защиты).
3. Браузер переходит на указанную категорию посуточной аренды Avito.
4. Программа собирает базовые данные всех объявлений на каждой странице каталога (ID, название, цена, ссылка, наличие бейджа «Мгновенная бронь», рейтинг хоста).
5. При наличии пагинации — автоматически переходит на следующие страницы.
6. После обхода каталога — программа заходит в каждую карточку объявления и извлекает детальные данные: координаты, календарь занятости на 60 дней (через datepicker с листанием месяцев), минимальный срок аренды (кликом на свободную дату в календаре), реальные цены на каждый свободный день (через датепикер: заезд → выезд → чтение цены). Если календарь не загрузился после 3 перезагрузок страницы — карточка считается проваленной (мусорные данные в БД не записываются). При обнаружении блокировки — программа ожидает до 5 раз с интервалом 15 секунд, а если не помогло и подключены прокси — автоматически меняет IP (до 3 смен на карточку). Плановая смена прокси происходит каждые N карточек (настраивается через ROTATE_EVERY_N_LISTINGS). Для экономии трафика блокируется загрузка картинок, шрифтов и аналитических скриптов.
7. Собранные данные сохраняются в SQLite (`data/avito_listings.db`).
8. Результаты экспортируются в Excel (`data/avito_report.xlsx`).

### Альтернативный запуск (если установлена как пакет)

```bash
avito-parser
```

## Результат — Excel-файл

После успешного запуска в папке `data/` появится файл `avito_report.xlsx` со следующими столбцами:

| Столбец | Описание |
|---|---|
| ID объявления | Уникальный идентификатор объявления (формат av_<id>) |
| Название | Оригинальное название из объявления |
| Категория жилья | Тип объекта (Комната, Студия, 1к, 2к, 3к, 4к+) |
| Широта | Географическая широта объекта |
| Долгота | Географическая долгота объекта |
| Средняя цена (руб./сут.) | Средняя цена за сутки по свободным дням |
| Занятость (%) | Процент занятых дней из ближайших 60 |
| Мин. срок (сут.) | Минимальный срок аренды в сутках |
| Мгновенное бронирование | Наличие мгновенного бронирования (Да/Нет) |
| Рейтинг хоста | Рейтинг хоста на Avito |
| Последнее обновление хостом | Дата последнего обновления календаря |
| Цены 60 дней | Массив цен на ближайшие 60 дней (компактный вид) |
| Календарь 60 дней | Массив занятости на 60 дней: 0 — свободен, 1 — занят |
| Ссылка | Прямая ссылка на объявление (кликабельная) |
| Дата снимка | Дата и время сбора данных |

### Как работать с файлом

1. Откройте файл в Excel или LibreOffice Calc.
2. Используйте автофильтр в столбце «Категория жилья» — выберите нужный тип (например, «1к»).
3. Отсортируйте по столбцу «Средняя цена» — увидите объекты от дешёвых к дорогим.
4. Отсортируйте по столбцу «Занятость (%)» — найдите наиболее/наименее востребованные объекты.
5. Сравните цены и занятость аналогичных объектов в одном районе.

### Принципы архитектуры

Проект построен на принципах SOLID:

- **Single Responsibility.** Каждый модуль отвечает за одну задачу: `browser_service` — за браузер, `scraper_service` — за обход каталога, `listing_service` — за парсинг карточки объявления, `export_service` — за Excel.
- **Open/Closed.** Компоненты расширяемы без модификации. Например, для добавления новой площадки (Суточно.ру) достаточно создать альтернативный `ListingService`, не меняя существующие сервисы.
- **Liskov Substitution.** Репозиторий определён как абстрактный класс (`BaseListingRepository`). SQLite-реализация может быть заменена на PostgreSQL или in-memory хранилище без изменения сервисов.
- **Interface Segregation.** Каждый интерфейс содержит только необходимые методы. Сервисы не зависят от лишних методов других модулей.
- **Dependency Inversion.** Сервисы зависят от абстракций, а не от конкретных реализаций. Все зависимости инжектируются через конструкторы.

### Используемые паттерны

- **Repository** — абстракция над хранилищем данных (SQLite, с возможностью замены на PostgreSQL).
- **Service Layer** — бизнес-логика в сервисах, координация между компонентами.
- **Factory** — фабричные функции в `__main__.py` для создания зависимостей.
- **Strategy** — `ListingService` можно заменить на альтернативную реализацию для другой площадки (Суточно.ру).
- **Decorator** — retry-декоратор для повторных попыток при сбоях.

## Устранение неполадок

### Ошибка «Обязательная переменная окружения не задана»

Убедитесь, что файл `.env` существует и содержит заполненное значение `AVITO_CATEGORY_URL`. Сверьтесь с `.env.example`.

### Браузер не запускается

Убедитесь, что Chromium установлен:

```bash
playwright install chromium
```

Если вы на Linux-сервере без графического интерфейса, установите `HEADLESS_MODE=true` в `.env`.

### Avito блокирует доступ

Программа использует stealth-настройки для обхода защиты, но иногда Avito может показать CAPTCHA или заблокировать доступ по IP. Программа автоматически обнаруживает блокировку и повторяет попытки:

- На странице каталога — до 10 попыток с интервалом 15 секунд.
- На странице карточки объявления — до 5 попыток с интервалом 15 секунд.
- При первоначальной навигации — до 5 повторных попыток после ожидания разблокировки.
Если автоматические повторы не помогают:

- Убедитесь, что HEADLESS_MODE=false (браузер с интерфейсом).
- При появлении CAPTCHA — решите её вручную в открытом окне браузера.
- Увеличьте PAGE_WAIT_TIME (например, до 60000).
- Уменьшите MAX_PAGES для снижения нагрузки.

Использование прокси
Для стабильного парсинга большого количества объявлений (100+) рекомендуется использовать residential прокси. Программа поддерживает автоматическую ротацию прокси:

- Создайте файл data/proxies.txt с прокси в формате host:port:user:pass (по одному на строку).
- Укажите путь в .env: PROXY_FILE_PATH=data/proxies.txt.
- Настройте частоту ротации: ROTATE_EVERY_N_LISTINGS=70 (рекомендуется 50–80).

Программа использует прокси с первого запуска (ваш реальный IP не используется). При бане на карточке — автоматически переключается на следующий прокси из пула (до 3 смен на карточку). Плановая ротация происходит каждые N карточек для предотвращения банов. Если PROXY_FILE_PATH пуст — парсер работает с вашим IP без прокси.

Добавьте proxies.txt в .gitignore, чтобы учётные данные прокси не попали в репозиторий.


### Excel-файл не создаётся

Убедитесь, что этап парсинга прошёл успешно — без собранных объявлений экспорт пропускается. Проверьте логи на наличие ошибок парсинга карточек.

### Программа работает медленно

- Уменьшите `PAGE_WAIT_TIME` (например, до `10000`).
- Ограничьте количество страниц через `MAX_PAGES`.
- Учитывайте, что программа заходит в каждую карточку объявления — при большом количестве объявлений парсинг может занять значительное время.

### Тестовый прогон (одно объявление)
Для быстрой проверки работоспособности парсера используйте скрипт на одно объявление:

python scripts/test_single_listing.py

Скрипт выполняет полный pipeline, но обрабатывает только первое объявление:

- Открывает браузер и переходит на каталог Avito.
- Парсит первую страницу каталога.
- Берёт первое объявление и заходит в его карточку.
- Извлекает все детальные данные (координаты, календарь, цены, мин. срок).
- Сохраняет в отдельную тестовую БД (data/test_single_listing.db).
- Экспортирует в отдельный тестовый Excel (data/test_single_listing.xlsx).
- Тестовые файлы не затрагивают основную базу и основной отчёт.

Тестовый прогон (30 объявлений, 3 параллельных потока)
Для полноценного тестирования с прокси и параллельной обработкой:

python scripts/test_10_listings.py

Скрипт запускает 3 параллельных воркера, каждый со своим прокси и браузерной сессией:

- Парсит каталог одним браузером (первый прокси) → берёт 30 объявлений.
- Разбивает на 3 группы по 10 и запускает 3 воркера параллельно.
- Каждый воркер: свой прокси → прогрев сессии (обход нейтральных страниц Avito) → парсинг 10 карточек с увеличенными паузами (8–15 сек) и имитацией поведения.
- Блокирует загрузку изображений, шрифтов и аналитических скриптов для экономии трафика.
- Failover: если хотя бы одна карточка не спарсилась (календарь не загрузился после всех попыток) — неспарсенные карточки передаются новому воркеру с новым прокси и новой сессией (до 3 failover-раундов).
- Сохраняет в отдельную тестовую БД (data/test_10_listings.db).
- Экспортирует в отдельный тестовый Excel (data/test_10_listings.xlsx).
- Тестовые файлы не затрагивают основную базу и основной отчёт.
Требования: минимум 3 прокси в файле proxies.txt (дополнительные используются для failover). Прокси расходуются последовательно: #1 — каталог, #2–#4 — основные воркеры, #5+ — failover.

## Повторный запуск

Программа поддерживает повторный запуск:

- **База данных** сохраняется между запусками. Повторный парсинг обновляет существующие объявления (по `external_id`) и добавляет новые.
- **Excel-файл** перезаписывается при каждом запуске и содержит все объявления из базы.

Это значит, что вы можете парсить разные категории (меняя `AVITO_CATEGORY_URL` в `.env`) и накапливать данные в одной базе. Учитывайте, что при смене категории (например, с посуточной аренды в СПб на Москву) все объявления будут в одной базе — при необходимости удалите файл `data/avito_listings.db` для чистого старта.
